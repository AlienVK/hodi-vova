# Ходи, Вова, ходи! 🚶

Веб-приложение (mobile-first) для соревнования по ежедневным шагам с виртуальным
маршрутом **Лондон → Париж**. Участники вручную вносят шаги, километры считаются
автоматически, прогресс отображается фишками на схематической карте. Первый
дошедший до Эйфелевой башни завершает круг — все фишки возвращаются на старт.

Реализовано по PRD v1 (`prd.md`). Чистый HTML/CSS/JS, без сборки и внешних
библиотек рендеринга — только облачный бэкенд (Firebase) для общих данных.

## Возможности

- **Карта** — вертикальный маршрут с 5 точками (Тауэрский мост → Кентерберийский
  собор → Белые скалы Дувра → Кале → Эйфелева башня), встроенные SVG-иллюстрации,
  водный участок Ла-Манш, фишки участников, номер круга и победитель прошлого круга.
- **Статистика** — рейтинг за месяц по шагам (с выбором месяца), диаграмма
  помесячной динамики, таблица помесячных итогов с переключателем шаги/км, блок
  текущего круга. Всё — собственными SVG/CSS.
- **Стена** — лента сообщений; писать могут вошедшие участники, читать — все.
- **Мои данные** — вход участника, ввод/редактирование шагов за дату, авторасчёт км.
- **Админ** — вход по мастер-паролю: регистрация/удаление участников, модерация стены.

## Режимы работы

Приложение определяет режим по `config.js`:

| Режим | Когда | Данные |
|---|---|---|
| **Демо (localStorage)** | `FIREBASE_CONFIG` пуст | Локально в браузере, не общие между устройствами (синхронизация только между вкладками) |
| **Облачный (Firebase)** | `FIREBASE_CONFIG` заполнен | Общие, живое обновление у всех в реальном времени |

Демо-режим удобен, чтобы сразу посмотреть приложение. Для реального соревнования
нужен облачный режим (см. ниже).

## Быстрый запуск локально

Файлы подключены обычными `<script>` — можно просто открыть `index.html` в браузере.
Если браузер ограничивает `file://`, поднимите любой статический сервер, например:

```bash
npx serve .          # если есть Node
# или
python -m http.server 5555
```

## Настройка Firebase (облачный режим)

1. Создайте проект на <https://console.firebase.google.com> (нужен аккаунт Google).
2. **Firestore Database** → создать базу (production mode).
3. **Authentication** → Sign-in method → включить **Anonymous**.
4. Project settings → «Ваши приложения» → Web → скопируйте объект конфигурации.
5. Вставьте его в `config.js` → `FIREBASE_CONFIG` и задайте `MASTER_PASSWORD`.
6. Firestore → Rules → вставьте содержимое `firestore.rules` и опубликуйте.

Параметры расчётов (длина шага, длина маршрута, лимит шагов, точки маршрута,
опциональные URL фото точек) — там же, в `config.js` (раздел 10.3 PRD).

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub и загрузите файлы проекта.
2. Settings → Pages → Source: ветка `main`, папка `/root` → Save.
3. Через минуту сайт доступен по публичному HTTPS-URL — делитесь ссылкой.

> ⚠️ **Безопасность (раздел 12 PRD).** В публичном репозитории `MASTER_PASSWORD`
> и код видны всем — это не настоящий секрет, а защита от «случайного» доступа к
> админ-функциям. Пароли участников хранятся в виде хеша с солью (уровень
> дружеского соревнования, не криптостойкий). Не используйте важные пароли и не
> храните в приложении конфиденциальные данные.

## Структура проекта

| Файл | Назначение |
|---|---|
| `index.html` | Каркас страницы, 5 вкладок |
| `config.js` | Единая конфигурация (заполнить перед развёртыванием) |
| `styles.css` | Тема «трансканальное путешествие», mobile-first |
| `icons.js` | SVG-иллюстрации точек и генератор аватара-монограммы |
| `auth.js` | Хеширование паролей (SHA-256 + соль) |
| `logic.js` | Бизнес-логика: км, прогресс, круги, месячные итоги, рейтинг |
| `store.js` | Слой данных: Firebase Firestore или localStorage |
| `app.js` | UI и связывание всего вместе |
| `firestore.rules` | Правила безопасности Firestore |

## Первые шаги после развёртывания

1. Откройте вкладку **Админ**, войдите по мастер-паролю.
2. Зарегистрируйте участников (логин + пароль, при желании — URL аватара).
3. Раздайте участникам их логины/пароли.
4. Участники входят во вкладке **Мои данные** и вносят шаги — карта и статистика
   обновляются автоматически.
